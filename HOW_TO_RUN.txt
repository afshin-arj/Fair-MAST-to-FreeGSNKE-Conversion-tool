HOW TO RUN (Top-level)
======================

Quickest start (interactive launcher)
-----------------------------------

Windows:
  run_pipeline.cmd

Linux/macOS:
  ./run_pipeline.sh

Both launchers prompt for one or more shot numbers ONLY (space/comma
separated, e.g. 30201 or 30201 30202). Everything else (download,
extraction, coil map, machine authority, FreeGSNKE inverse+forward execution)
comes from the shipped configs/default.json.

Outputs land under:
  SHOT/<shot_number>/     e.g. SHOT/30201/

On Windows, run_pipeline.cmd always pauses at the end (success or failure)
so you can read the console. Set RUN_PIPELINE_NO_PAUSE=1 to skip the pause.

Launcher install hygiene:
  Set RUN_PIPELINE_SKIP_INSTALL=1 to skip pip upgrade+install entirely.
  Otherwise the launchers reinstall only when pyproject.toml changed since the
  last install (SHA256 marker at .venv/.install_marker).
  Launchers also bootstrap tools/s5cmd (scripts/ensure_s5cmd.py) and
  .venv-freegsnke (scripts/ensure_freegsnke_env.py). Skip the FreeGSNKE env
  step with RUN_PIPELINE_SKIP_FREEGSNKE_ENV=1.

Download cache reuse (allow_cache_reuse=true in configs/default.json):
  Re-running a shot reuses non-empty data_cache/shot_<N>/<group>.zarr trees
  instead of re-syncing. When every required group is cached the run skips the
  network entirely; per-group cache hits, resolved S3 paths, and file
  counts/bytes are recorded in the manifest ("download_report").

Manual install + CLI:

pip install -e .
pip install -e ".[zarr]"   # optional, enables extraction

mast-freegsnke doctor --config configs/default.json
mast-freegsnke check --shot 30201 --config configs/default.json
mast-freegsnke run --shot 30201 --config configs/default.json

Batch mode (worst exit code; summary at the end; set batch_abort_on_failure
=true in config to stop at the first failing shot):

mast-freegsnke run --shots 30201 30202 --config configs/default.json

Optional: execute FreeGSNKE scripts after generation (requires freegsnke installed in the selected Python environment):

  # One-time FreeGSNKE env (Python 3.11; 3.14 may lack scipy wheels):
  py -3.11 -m venv .venv-freegsnke
  .venv-freegsnke\Scripts\python -m pip install -r requirements-freegsnke.txt

  # One-time s5cmd (if not on PATH):
  python scripts/ensure_s5cmd.py

  # Build machine_authority from a downloaded FAIR-MAST shot cache:
  # (1) probe/coil JSON authority from Level-2
  python scripts/build_authority_from_fairmast.py --shot-cache data_cache/shot_30201 --out machine_authority --shot 30201
  # (2) classic MAST FreeGSNKE pickles (NOT MAST-U) from Level-2 filaments + wall.zarr EFIT limiter
  python scripts/build_classic_mast_machine.py --shot-cache data_cache/shot_30201 --out machine_authority --shot 30201 --validate-tokamak
  # Optional archive-only: FreeGSNKE public MAST-U-like pickles (not production)
  # python scripts/fetch_freegsnke_machine.py

  mast-freegsnke run --shot 30201 --config configs/default.json

If FreeGSNKE lives in a different venv/conda env, point to that python:

  mast-freegsnke run --shot 30201 --config configs/default.json \
    --freegsnke-python /path/to/python

After a run, you can recompute the time window / QC / consensus:
  mast-freegsnke window --shot 30201 --config configs/default.json
  mast-freegsnke windowqc --shot 30201 --config configs/default.json
  mast-freegsnke consensus --shot 30201 --config configs/default.json

Deterministic window override (optional; not part of the happy path):
  mast-freegsnke run --shot 30201 --config configs/default.json --tstart 0.120 --tend 0.280

Machine / probe geometry authority
----------------------------------
FAIR-MAST publishes classic MAST (not MAST-U). Production defaults use
`machine_authority/` with FreeGSNKE pickles built from Level-2 PF filaments
(active circuits: Solenoid, P2_inner, P2_outer, P3–P6) plus wall.zarr EFIT
limiter geometry (not CAD vessel; passives empty — pf_passive has no ρ).
Active-coil resistivity is FreeGSNKE copper default 1.55e-8 (declared).
Templates with CHANGE_ME / TEMPLATE markers are refused. Do not invent
metrology. When geometry resolves, the run writes:
  SHOT/<N>/magnetic_probes.pickle
  SHOT/<N>/magnetic_probes.json

Diagnostic contract metrics (enabled by default in configs/default.json)
------------------------------------------------------------------------
As of v10.4.0 the shot-only happy path extracts experimental per-probe traces,
solves FreeGSNKE at multiple deterministic window sample times, emits
multi-row synthetic probe CSVs, and scores residuals under an explicit
contracts authority:

  Experimental (from FAIR-MAST Level-2 magnetics.zarr):
    inputs/flux_loops.csv   <- flux_loop_flux (flux_loop_channel x time, Wb)
    inputs/pickups.csv      <- b_field_pol_probe_{ccbv,obv,obr}_field (T)
    Column names = FAIR-MAST channel coordinate values verbatim.

  Synthetic (from FreeGSNKE inverse solves via tokamak.probes):
    synthetic/synthetic_fluxloops.csv
    synthetic/synthetic_pickups.csv
    Column names = geometry-authority probe names; ONE ROW PER SOLVED TIME.
    Sample times: metrics_timebase execution authority, rule
    linspace_window_inclusive with n_times (config key metrics_n_times,
    default 5) equally spaced samples in the finalized window
    [t_start, t_end]. Preferred path (v10.5.0): full FreeGSNKE inverse
    at each sample (fresh Inverse_optimizer, continuation seeding,
    declared max_solving_iterations + per_time_timeout_s). solve_mode
    is typically full_inverse; forward_gs_at_measured_pf_ip is only an
    explicit per-time fallback. Per-time status is recorded in
    synthetic/synthetic_times.json and in the manifest. The single t0
    inverse solve remains the source of inverse_dump.pkl / plots /
    forward replay.

  Authority:
    configs/diagnostic_contracts.json
    (identity sign=1 / scale=1.0; flux loops: FL_<channel with / -> _>;
     pickups: verbatim; units from zarr attrs; ALL 93 identity-mapped
     channels contracted -- no per-shot NaN exclusions)

  Metrics:
    interpolate experimental onto the solved synthetic times; score
    RMS/MAE/max_abs across all sample times; write
    SHOT/<N>/metrics/reconstruction_metrics.json and residual_*.csv.
    Channels that are all-NaN on the current shot are skipped shot-scoped
    (status skipped_all_nan; n_skipped_all_nan in the summary) -- not a
    blocking failure, never a fabricated zero. Real failures remain
    blocking when enable_contract_metrics=true.

Excluded by default (honestly, with evidence): mirnov / saddle / omaha families.
Level-2 attrs give no published V->T / V->Wb factors; some variables have
self-contradictory unit metadata (units='T' vs label='Tesla/sec' or 'mT').
Traces ARE extracted verbatim for audit under
SHOT/<N>/inputs/audit_other_timebase/ with per-variable evidence in
extract_meta.json.

Optional promotion (v10.6.0) — diagnostic calibration authority:
  configs/diagnostic_calibration.json
    status=awaiting_authority, channels={} by default (no invented scales).
  Populate channels with explicit scale/sign/source (and unit_resolution
  when resolving units-vs-label). Calibrated families write to
  inputs/mirnov.csv | saddle.csv | omaha.csv. FreeGSNKE identity contracts
  merge only for synthesize=true point pickups with units_out T|Wb and a
  geometry syn_probe. Saddle has no FreeGSNKE synthesizer; OMAHA has no R/Z
  in machine_authority. See configs/diagnostic_calibration.example.json.
  Doctor / launcher print one status line for this authority.

Validate without a run:
  mast-freegsnke contracts-validate --contracts configs/diagnostic_contracts.json
  mast-freegsnke doctor --config configs/default.json

Outputs (on a successful shot run):
  SHOT/<N>/contracts/diagnostic_contracts.resolved.json
  SHOT/<N>/synthetic/synthetic_fluxloops.csv
  SHOT/<N>/synthetic/synthetic_pickups.csv
  SHOT/<N>/synthetic/synthetic_times.json
  SHOT/<N>/metrics/reconstruction_metrics.json and residual_*.csv
  SHOT/<N>/inputs/audit_other_timebase/*.csv (audit-only families)

Reruns of the same shot
-----------------------
Rerunning a shot archives the previous run into SHOT/<N>/history/<timestamp>/
before writing fresh outputs, so the top level of SHOT/<N>/ is always one
clean, auditable run. Nothing is deleted.
